5) Board-to-Panel Association Framework (New)
- Build a GUI workflow to map physical boards (Arduino or other serial devices) to logical cockpit panels.
- Add a board discovery layer:
  - enumerate connected serial devices
  - collect stable identity hints (COM port, VID/PID when available, optional device-reported name)
  - track reconnects and port renumbering behavior
- Add panel mapping UI:
  - create/edit panel profiles
  - assign one or more boards to each panel
  - persist mappings to config file for automatic restore on startup
- Add communication verification tools:
  - per-board heartbeat/handshake status
  - send test command and verify response/roundtrip
  - visual status indicators (Connected / Degraded / No Data)
- Add auto-association strategy (phase 1):
  - exact match on persisted board identity
  - fallback to user confirmation if ambiguous
  - never silently remap when identity confidence is low

Acceptance:
- User can connect multiple boards and map them to panels through GUI without manual script edits.
- Mappings are restored automatically after restart and after COM port number changes when identity is known.
- Verification view can confirm communication health per board before mission start.
- Ambiguous hardware matches require explicit user confirmation.
