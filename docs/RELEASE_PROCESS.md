# Hornet Link Release Process

This repository now includes automated release packaging via GitHub Actions.

## Triggering a release

1. Ensure `main` is green in CI.
2. Create and push a version tag:

```bash
git tag v0.1.0
git push origin v0.1.0
```

Pushing a `v*` tag triggers `.github/workflows/release.yml`.

## What the release workflow publishes

For each tag release, GitHub Actions builds and uploads:

- `hornet-link-Release.zip` (runtime bundle from `build-package.cmd`)
- `hornet-link-<version>-Windows.zip` (CPack ZIP from `make-installer.cmd ZIP`)
- `SHA256SUMS.txt` (checksums for all uploaded archives)

## Notes

- Release notes are auto-generated by GitHub (`generate_release_notes: true`).
- The workflow currently targets `windows-latest` and Release configuration only.
- NSIS `.exe` installer generation remains optional and is not required for releases.
